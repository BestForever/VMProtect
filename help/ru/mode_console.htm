Консольная версия

После создания проекта в GUI режиме в дальнейшем может использоваться консольная версия (VMProtect_Con.exe), вызов которой осуществляется следующим образом:

VMProtect_Con Файл [Выходной файл] [-pf Файл проекта] [-sf Файл скрипта] [-lf Файл с параметрами лицензирования] [-bd Дата сборки (yyyy-mm-dd)] [-wm Название водяного знака] [-we]

  • Файл - имя исполняемого файла, который необходимо защитить (*.exe, *.dll и т.д.), или имя файла проекта (*.vmp). Если в качестве исходного файла указан файл проекта, то имя исполняемого файла будет браться из файла проекта.
  • Выходной файл - имя и путь к защищенному файлу, который должен быть создан после обработки исходного файла. Если параметр не задан, его значение берется из файла проекта.
  • Файл проекта - имя и путь к файлу проекта, созданного в GUI режиме. Если параметр не задан, поиск файла проекта *.vmp будет осуществляться в папке с исполняемым файлом.
  • Файл скрипта - имя файла, содержащего скрипт, с помощью которого будет обработан защищаемый файл. Если параметр не задан, будет использоваться скрипт из текущего файла проекта.
  • Файл с параметрами лицензирования - имя файла, содержащего параметры лицензирования. Если параметр не задан, будут использовать параметры лицензирования из текущего файла проекта.
  • Дата сборки - дата сборки приложения в формате "yyyy-mm-dd". Если параметр не задан, будет использоваться текущая дата. Дата сборки записывается в защищаемое приложение и используется системой лицензирования при проверке серийных номеров, содержащих поле "Максимальная дата сборки".
  • Название водяного знака - название водяного знака, который будет вставлен в защищаемый файл. Если название водяного файла не указано, в защищенный файл будет вставлен водяной знак, который был указан при создании проекта.
  • we - при использовании данного параметра все предупреждения будут отображаться как ошибки.
